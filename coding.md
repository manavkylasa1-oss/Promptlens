# PromptLens Examples — Coding & Technical Prompts

---

## Example 1 — Build a Feature

**❌ Before:**
```
add a search bar to my website
```

**✅ PromptLens Improved:**
```
Add a search bar to an existing HTML/CSS/JavaScript website.

Context:
- The site is a static blog with articles stored as HTML files
- No backend or database — everything is client-side
- I'm using vanilla JS, no frameworks

Requirements:
- Search bar appears in the navigation header
- Filters articles in real-time as the user types (searches by title only)
- Articles not matching the query are hidden (not deleted from DOM)
- Matches are case-insensitive
- Works on mobile (responsive)

Please:
- Write clean, commented code
- Show me exactly where to insert each snippet in my existing structure
- Explain in one sentence what each function does
```

## Example 2 — Debug an Error

**❌ Before:**
```
my code isn't working fix it
```

**✅ PromptLens Improved:**
```
Here is my Python code and the exact error message I'm getting. Please help me fix it.

[PASTE CODE HERE]

Error message:
[PASTE ERROR HERE]

What I was trying to do: [describe in one sentence]
What I've already tried: [list any fixes you attempted]

Please:
1. Identify the cause of the error
2. Explain why it's happening in plain English
3. Show me the fixed code
4. Tell me how to avoid this type of error in future
```

## Example 3 — Explain a Concept

**❌ Before:**
```
explain APIs
```

**✅ PromptLens Improved:**
```
Explain what an API is to someone who understands basic web browsing but has
never coded before.

Use a real-world analogy first, then explain the technical concept.
Cover:
- What an API is and what it does
- Why developers use them (one practical example)
- What "calling an API" means in plain English

Keep it under 200 words. No code examples — just concepts.
```

## Example 4 — Code Review

**❌ Before:**
```
review my code
```

**✅ PromptLens Improved:**
```
Please review the following [language] code and give me structured feedback.

[PASTE CODE HERE]

I want feedback on:
1. Correctness — does it do what it's supposed to?
2. Efficiency — any performance improvements?
3. Readability — is it easy to understand and maintain?
4. Security — any obvious vulnerabilities? (especially input validation)

My experience level: intermediate
Be direct — I want honest feedback, not just praise.
If something is fine, say so briefly. Spend more time on what needs improvement.
```
